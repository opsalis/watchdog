How it works
Simple, distributed, yours.
1

Start the master

One Docker container runs the master server. It stores jobs in SQLite and listens for worker connections on a WebSocket endpoint.

2

Add workers anywhere

Deploy worker containers on any server, cloud region, or Raspberry Pi. Each worker connects to the master and receives check jobs automatically.

3

Register a URL

POST one JSON request to create a monitoring job. Set interval, expected status, timeout, and alert webhook. Results flow back in real time.

Install
Running in under two minutes.

Clone the repository, then start everything with Docker Compose. Two workers start automatically.

# Clone
git clone https://github.com/opsalis/watchdog.git
cd watchdog

# Start master + 2 workers
docker compose up -d

# Create your first monitoring job
curl -X POST http://localhost:4001/v1/jobs \
  -H "Content-Type: application/json" \
  -d '{"url":"https://yoursite.com","interval_seconds":60,"alert_email":"you@example.com"}'

API reference
Five endpoints. That's it.

Create a job

POST /v1/jobs

{
  "url": "https://example.com",
  "interval_seconds": 60,
  "locations": ["us-east", "eu-west"],
  "alert_email": "ops@example.com",
  "expect_status": 200,
  "timeout_ms": 10000
}

Job status + uptime

GET /v1/jobs/:id

{
  "id": "abc-123",
  "url": "https://example.com",
  "stats": {
    "uptime_pct": "99.94",
    "avg_latency": 87,
    "total_checks_24h": 1440
  }
}

List all jobs

GET /v1/jobs

[{ "id": "...", "url": "...", ... }]

Check history

GET /v1/jobs/:id/checks?limit=100

[{
  "up": 1,
  "latency_ms": 42,
  "location": "eu-west",
  "checked_at": 1711800000000
}]
